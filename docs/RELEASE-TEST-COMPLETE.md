| 시나리오 | 검증 항목 |
|---------|----------|
| 강제 업데이트 | ✅ 팝업 표시<br>✅ "나중에" 없음<br>✅ 매번 표시<br>✅ 뒤로가기 차단 |
| 선택적 업데이트 | ✅ 팝업 표시<br>✅ "나중에" 있음<br>✅ 추적됨<br>✅ 새 버전 시 재표시 |

### Notice Policy
| 시나리오 | 검증 항목 |
|---------|----------|
| 공지 표시 | ✅ 팝업 표시<br>✅ 추적됨 |

| emergency > update | ✅ emergency만 표시 |

## 🎯 핵심 검증 포인트

### Emergency
- ⭐ **추적 없음**: 매번 표시되는가?
- ⭐ **X 버튼 제어**: is_dismissible이 작동하는가?
- ⭐ **최우선순위**: 다른 팝업을 무시하는가?

### Update
- ⭐ **강제/선택 구분**: "나중에" 버튼이 올바른가?
- ⭐ **추적**: "나중에" 후 재표시 안 되는가?
- ⭐ **버전 변경**: 새 버전 시 재표시되는가?

### Notice
- ⭐ **버전 관리**: 오타 수정 시 재표시 안 되는가?
- ⭐ **버전 증가**: 새 공지 시 재표시되는가?
- ⭐ **추적**: X 클릭 후 재표시 안 되는가?

### 우선순위
- ⭐ **순서**: emergency > update > notice
- ⭐ **무시**: 상위 팝업 있으면 하위 무시
